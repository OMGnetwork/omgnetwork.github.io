Run Watcher Locally

At the end of this guide you should know how to run your own Watcher locally.

Requirements

  1. Docker Compose. This is the recommended method of starting the blockchain services, with the auxiliary services automatically provisioned through Docker.

To check if you have Docker Compose installed, run the following command in your terminal:

docker-compose --version

Example output:

docker-compose version 1.25.5, build 8a1c60f6
  1. A fully synced Ethereum client. There are several options to choose from:
  • Run Geth client.
  • Run Parity client.
  • Use one of external services: Infura, QuickNode, Rivet, etc.

Supported Platforms

Watcher currently supports the following operating systems:

  • Ubuntu 16.04
  • Ubuntu 18.04
  • MacOS ?

Step 1 - Clone repository

Clone elixir-omg:

git clone https://github.com/omisego/elixir-omg.git

Step 2 - Check TCP ports

Before attempting the start up please ensure that you are not running any services that are listening on the following TCP ports: 9656, 7434, 7534, 5000, 8545, 5432, 5433. All commands should be run from the root of the repo.

You can use lsof, netstat, nmap or other alternatives to accomplish that.

Linux:

sudo lsof -i -P -n | grep LISTEN
sudo netstat -tulpn | grep LISTEN
sudo nmap -sTU -O IP-address-Here

The netstat command has been deprecated for some time on Linux. Therefore, you can use the ss command as follows:

sudo ss -tulw
sudo ss -tulwn

MacOS:

netstat -anp tcp | grep LISTEN

Step 3 - Mofidy configurations

Change ETHEREUM_RPC_URL..

Step 4 - Run a Watcher instance

To run a Watcher instance, you need to start the required docker containers. The parameter -d allows running containers in the background.

docker-compose -f docker-compose-watcher.yml up -d

To see logs, use the following command:

docker-compose logs -f

Once the Ethereum client is running, you can use Ctrl + P, Ctrl + Q to detach from the container without stopping it. You will need to leave the container running for the Watcher node to connect to it.

If the container was stopped and you need to run it again, you can simply use the following command:

docker start -i eth
